현재 브랜치의 git log와 diff를 분석해서 GitHub PR description을 한국어로 작성해줘.

아래 형식을 반드시 따라줘:

---

## 개요
[무엇을 왜 구현했는지 1-3줄. 설계 의도나 배경이 있으면 포함]

## 주요 변경사항

### [이모지] [카테고리명]
- 핵심 변경사항 설명
- 설계 결정과 이유
- 코드 스니펫은 설계 의도가 코드로 표현될 때만 포함

(카테고리는 변경사항에 맞게 자유롭게 구성. 이모지 예시: 📦 엔티티, 🌐 API, 🔐 인증, 🔍 성능, ✅ 테스트, ⚙️ 설정)

## 가이드 (해당되는 내용이 있을 때만 포함)

### API 사용 예시 (새로운 API가 있을 때)
[요청/응답 예시]

### 알아두어야 할 점 (특이한 요구사항, 제약사항, 사이드이펙트, 운영 시 주의사항 등)
- ...

### 상태 전이 (상태 모델이 새로 생기거나 변경될 때)
[다이어그램 또는 테이블]

## 다음에 해야 할 작업
- [ ] ...

---

규칙:
- 파일 목록 나열 금지
- P0/P1/P2 우선순위 표기 금지
- 기술적 결정은 "주요 변경사항" 안에 녹여서 설명
- "가이드" 섹션 내 소제목은 내용에 따라 필요한 것만 포함
- "가이드" 섹션 자체가 불필요하면 생략
